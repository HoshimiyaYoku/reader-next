# 章节摘要栏

章节摘要栏是在阅读页旁边显示的轻量 AI 辅助。它只服务当前章节：给一段摘要，再给一组后续阅读需要记住的要点。

::: tip 设计取向
它不并入 AI 资料。AI 资料负责整本书的长期结构化记忆；章节摘要栏负责当前阅读时的即时回顾。
:::

## 入口

在阅读页打开摘要栏后，可以查看当前章节的摘要和要点。隐藏后，Reader Next 会记住这个状态；刷新页面不会强行重新显示。

## 内容结构

- **摘要正文**：简短说明本章主线。
- **要点列表**：人物、关系、地点、目标、物品或已揭示信息。
- **前文上下文**：生成时会参考前面章节的缓存摘要，减少长篇断层。

要点列表使用同一块内容承载，不再每条都放进单独高亮框，阅读干扰更少。

## 生成控制

设置里保留必要控制：

| 项目 | 说明 |
| --- | --- |
| 功能启用 | 是否启用章节摘要功能 |
| 自动生成 | 进入章节后是否自动尝试生成摘要 |
| 详细程度 | 控制摘要长度和要点数量 |
| 最大字数 | 给模型的输出上限 |

详细程度实际参与 prompt：

- **短**：摘要控制更短，要点约 2-4 条。
- **正常**：摘要中等，要点约 3-6 条。
- **详细**：摘要更完整，要点约 5-8 条。

点击重新生成前会先保存当前设置。

## 模型配置

章节摘要使用现有文本模型配置。管理员可以在 AI 相关设置中配置后端模型，普通用户在有权限时直接使用后端配置。

如果使用自定义网关，文本接口路径可以按 provider 调整，例如：

- `/v1/chat/completions`
- `/v1/responses`
- `/v1/messages`
- Gemini `:generateContent` 转发入口

## 数据边界

- 摘要按用户、书籍和章节隔离保存。
- 摘要缓存不写入 AI 资料 memory。
- 隐藏/显示偏好保存在阅读配置里。
- 自动生成会避开已隐藏摘要栏的场景，避免刷新后打扰阅读。
