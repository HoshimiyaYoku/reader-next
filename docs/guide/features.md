# 功能特性

Reader Next 当前重点覆盖书源阅读、服务端书架、本地书籍、跨设备进度、章节摘要、AI 资料和 AI 追赶。

## 阅读器

- 阅读页目录、书签、阅读设置、TTS、自动阅读。
- 亮色/暗色主题和移动端阅读交互。
- 打开书籍前同步服务端最新进度。
- 恢复上次阅读会话时优先使用服务端进度。
- 章节正文缓存和批量缓存。

## 书源解析

- CSS Selector
- JSONPath
- XPath
- Regex
- JavaScript
- Legado 风格规则兼容
- 搜索、详情、目录、正文多阶段解析

## 服务端书架

- 添加、删除、分组、排序书籍。
- 最近阅读和阅读进度保存。
- SQLite 存储用户、书源、书架、阅读进度和 AI 数据。
- 文件缓存保存章节内容和上传资源。

## 本地书籍

本地书籍会进入服务端书架，而不是只保存在某台设备的浏览器里。

- 支持上传 `.txt`、`.epub`、`.mobi`、`.pdf` 格式文件。
- 自动解析常见章节标题（TXT）或提取目录结构（EPUB）。
- 生成稳定的本地书籍 URL 和章节 URL。
- 支持服务端目录、正文读取和阅读进度保存。
- 删除书籍时清理导入文件。
- 本地书籍不显示远程章节缓存标签。

## 章节摘要栏

- 阅读页侧边摘要栏展示本章摘要和要点。
- 支持自动生成、手动重新生成和隐藏状态持久化。
- 摘要要点合并在同一块内容中，避免每条都变成高亮卡片。
- 详细程度支持短、正常、详细，并影响 prompt 中的长度和要点数量。
- 设置入口集中到阅读器内，删掉重复开关。

更多说明见 [章节摘要栏](./chapter-summary)。

## AI 资料

- 按已读章节整理剧情概览、世界观、角色、关系和地点。
- 支持图片地图；图片生成失败时降级为关系图。
- 支持后端代理模型请求，避免浏览器跨域或暴露服务端密钥。
- AI 资料只处理已读范围，适合长篇阅读回顾。

更多说明见 [AI 资料](./ai-book)。

## AI 追赶

- 后台批量处理已读章节的 AI 资料整理。
- 支持暂停/恢复，前端可关闭后继续处理。
- 每完成一章立即保存，服务器重启只丢失任务状态，不丢失已完成资料。
- 进度条和当前章节实时显示。

## AI provider preset

- 文本模型 Base URL 和接口路径分开配置。
- 默认路径为 `/v1/chat/completions`。
- 可接入 `/v1/responses`、`/v1/messages`、Gemini `:generateContent` 等兼容转发入口。
- 后端配置可由管理员保存，普通用户只使用可见模型信息。

## RSS 和维护

- RSS 源管理、文章列表和文章阅读。
- WebDAV 备份/恢复书架、书源、阅读进度等数据。
- 服务端文件备份、下载、上传和删除。
- 书源导入导出。
